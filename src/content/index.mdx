# Overview

Iron.sh gives AI coding agents hardened dev environments: real machines with controlled network access, transparent secrets proxying, full egress audit logs, and instant snapshot/rollback.

AI coding agents install packages, run builds, and call APIs. That means real network access, real credentials, and real exposure to prompt injection, data exfiltration, and unauthorized outbound calls. Iron.sh is the infrastructure layer that makes this safe by default.

## Features

**Secrets proxying.** Agents never see real credentials. You store secrets in Iron.sh and they're injected at the network layer when the agent makes an outbound request. Inside the environment, the agent only sees a proxy token. It can't log, leak, or exfiltrate the real key.

**Default-deny egress.** Every outbound connection is policy-controlled. You allowlist the domains an agent needs (package registries, APIs, git hosts) and everything else is blocked. Rules can also be set to `warn` mode to audit traffic before locking things down.

**Snapshot, rollback, and fork.** Checkpoint your environment before a risky run. If the agent breaks something, restore in seconds. Fork a snapshot to try a different approach in parallel.

**Full audit logging.** Every outbound connection is logged with host, path, method, response code, bytes transferred, and which secrets were injected. You know exactly what your agent did.

**Persistent, long-running environments.** No 24-hour walls. Environments persist for hours, days, or weeks. Installed packages, config, and working state survive restarts. Your agent picks up where it left off.

## Quickstart

Before you begin, make sure you're running macOS or Linux. Windows is not currently supported.

<Steps>

### Step 1: Install the CLI

The quickest way to install `irons` is via the install script. It downloads the latest release, validates its SHA-256 checksum, and places the binary on your `$PATH`:

```sh
curl -fsSL https://install.iron.sh | bash
```

### Step 2: Onboard

Next, onboard with a single command:

```sh
irons onboard
```

This command will create an iron.sh account and API key for you, create GitHub and coding harness secrets, and provision a new [agent](./getting-started/coding-agents.mdx) for you to use.

![Image of irons onboard](/images/tui-onboard.png)

### Step 3: Use your Agent

You're now ready to use your agent! Your GitHub repo is automatically cloned, so you can get started right away. The previous command will drop you into a TUI for interacting with your agent, and managing egress rules. If it didn't, run `irons agents attach <agent-name>`.

![Image of default TUI screen](/images/tui-default.png)

To monitor your agent's egress attempts and allow/deny them in real time, press `Ctrl-]` to flip over to the egress monitoring pane:

![Image of default TUI screen](/images/tui-egress.png)

### Step 4: Tear Down the Agent

When you're done, destroy the agent and clean up all associated resources:

```sh
irons agents destroy <agent-name>
```

</Steps>
