# CLI Reference

The `irons` CLI is the primary interface for managing Iron.sh sandboxes. All commands require the `IRONS_API_KEY` environment variable to be set.

```sh
export IRONS_API_KEY=your-api-key
```

---

## `irons create`

Provisions a new sandbox and waits until it reaches the `ready` state.

```sh
irons create <name> [flags]
```

**Arguments**

| Argument | Description                          |
| -------- | ------------------------------------ |
| `name`   | A unique name for the sandbox        |

**Flags**

| Flag        | Default                  | Description                                                              |
| ----------- | ------------------------ | ------------------------------------------------------------------------ |
| `--key`     | `~/.ssh/id_rsa.pub`      | Path to the SSH public key to authorize on the sandbox                   |
| `--async`   | `false`                  | Return immediately without waiting for the sandbox to reach `ready`      |

**Examples**

```sh
# Create a sandbox with the default SSH key
irons create my-sandbox

# Create with a specific SSH key and don't wait
irons create agent-run-42 --key ~/.ssh/agent.pub --async
```

## `irons list`

Lists all sandboxes in your account with their current status.

```sh
irons list
```

**Example output**

```
NAME              STATUS    CREATED
my-sandbox        ready     5 minutes ago
agent-run-42      ready     2 hours ago
old-sandbox       stopped   3 days ago
```

---

## `irons status`

Prints the current status and connection details for a sandbox.

```sh
irons status <name>
```

**Arguments**

| Argument | Description           |
| -------- | --------------------- |
| `name`   | Name of the sandbox   |

---

## `irons ssh`

Opens an interactive SSH session into a sandbox.

```sh
irons ssh <name> [flags]
```

**Arguments**

| Argument | Description           |
| -------- | --------------------- |
| `name`   | Name of the sandbox   |

**Flags**

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--command` | Print the SSH command instead of executing it (useful for scripts or agents) |

**Examples**

```sh
# Open an interactive session
irons ssh my-sandbox

# Print the SSH command for use in a script or agent
irons ssh --command my-sandbox
# e.g. ssh -p 32847 user@host.iron.sh
```

---

## `irons destroy`

Destroys a sandbox and cleans up all associated resources. This is irreversible.

```sh
irons destroy <name> [flags]
```

**Arguments**

| Argument | Description           |
| -------- | --------------------- |
| `name`   | Name of the sandbox   |

**Flags**

| Flag      | Description                                              |
| --------- | -------------------------------------------------------- |
| `--force` | Stop the sandbox before destroying if it is still running |

**Examples**

```sh
irons destroy my-sandbox
irons destroy --force my-sandbox
```

## `irons egress`

Manages outbound network traffic rules for your sandboxes.

### `irons egress mode`

Sets the global egress enforcement mode.

```sh
irons egress mode <mode>
```

| Mode   | Behavior                                                          |
| ------ | ----------------------------------------------------------------- |
| `warn` | All outbound connections are permitted, but violations are logged |
| `deny` | Outbound connections to non-allowlisted domains are blocked       |

```sh
irons egress mode warn
irons egress mode deny
```

### `irons egress allow`

Adds a domain to the egress allowlist.

```sh
irons egress allow <domain>
```

```sh
irons egress allow api.openai.com
irons egress allow pypi.org
irons egress allow api.github.com
```

### `irons egress deny`

Removes a domain from the egress allowlist.

```sh
irons egress deny <domain>
```

```sh
irons egress deny pypi.org
```

### `irons egress list`

Lists all domains currently on the egress allowlist and the active mode.

```sh
irons egress list
```

**Example output**

```
MODE: deny

ALLOWED DOMAINS
api.openai.com
api.github.com
pypi.org
```

## `irons audit`

### `irons audit egress`

Displays the egress audit log for a sandbox — every outbound connection attempt, whether allowed or denied.

```sh
irons audit egress <name> [flags]
```

**Arguments**

| Argument | Description           |
| -------- | --------------------- |
| `name`   | Name of the sandbox   |

**Flags**

| Flag       | Description                                     |
| ---------- | ----------------------------------------------- |
| `--follow` | Stream new log entries in real time (like `tail -f`) |

**Examples**

```sh
# Print the full audit log
irons audit egress my-sandbox

# Stream new entries as they arrive
irons audit egress my-sandbox --follow
```

**Example output**

```
TIMESTAMP              DOMAIN                  STATUS
2025-01-15T10:42:01Z   api.openai.com          allowed
2025-01-15T10:42:03Z   pypi.org                allowed
2025-01-15T10:42:07Z   example.com             denied
```

## Global Flags

These flags are available on every command.

| Flag        | Description                                        |
| ----------- | -------------------------------------------------- |
| `--help`    | Print help for a command                           |
| `--version` | Print the current `irons` version                  |
| `--api-key` | API key to use instead of `$IRONS_API_KEY`         |

| Variable         | Description                           |
| ---------------- | ------------------------------------- |
| `IRONS_API_KEY`  | Your Iron.sh API key (**required**)   |
